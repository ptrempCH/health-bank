REST API for Accessing the Server


There are quite a few REST services running on the Tomcat Server. Each of the services as described later on are defined in a Java Servlet. The task of these services is to interact with the MongoDB database in the background. The services talk via JSON data to make life easier for JavaScript developers as well as for talking to the database.
We grouped similar services together and present one after the other in the following chapters:


  • User: In this group we discuss access to the user profile, changing circle and space associations for a user as well as making queries on users.
  • Authentication: In this group we discuss registering, logging in and out for users and institutes. The more we discuss access tokens for applications.
  • Records: Here we discuss adding records to the system and querying them again.
  • Applications: This group deals with adding, editing applications, users installing them and asking queries on them.
  • News: Dealing with news entries is discussed in this group.
  • Messages: Sending and receiving messesges between users and institutes is talked about here.
  • File: This group deals with files such as images, x-ray etc.
