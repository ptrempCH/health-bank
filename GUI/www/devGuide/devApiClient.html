JavaScript API for Accessing the Client functionality


In order to meet all the different REST services running on the Tomcat Server and for plenty of other things we provide many JavaScript functions that you can use for your applications and visualizations. Each of the functions is described later in one of the different categories. For all the functions that involve AJAX calls to the server there is the callback parameter which allows you to defined an onSuccess and an onError function which gets called respectively.
We grouped similar functions together and present one after the other in the following chapters:


  • User: In this group we discuss access to the user profile, changing circle and space associations for a user as well as making queries on users.
  • Authentication: In this group we discuss registering, logging in and out for users and institutes. The more we discuss access tokens for applications.
  • Records: Here we discuss adding records to the system and querying them again.
  • Applications: This group deals with adding, editing applications, users installing them and asking queries on them.
  • News: Dealing with news entries is discussed in this group.
  • Messages: Sending and receiving messesges between users and institutes is talked about here.
  • Files: This group deals with files and images such as user icons etc.
  • Other functions: Here we define a few additional functions which did not fit into one of the above categories


Almost all of the client API functions will have a parameter called 'callback' as you will certainly notice by clicking thorugh the different categories. This callback parameter allows the caller to set an 'onSuccess' and 'onError' function which will be called, when the asynchonous call to the server returns depending on if there was an error or not. If the user is not logged in anymore none of these functions will be called and the user is redirected to the login page.
