REST API for Files


The files category contains so far only one servlet. The /image or /Image servlet is used to retrieve images from the server which are stored in the database itself like e.g. the user icons or the application icons. Since we only want to get data from this service, just GET requests are allowed. This requests is best described with its JavaDoc:



The name parameter is the name of the image we like to retrieve. If we know the id of the icon, we can query by this value. The returned data is of the type of the image and contains the actual image data. It can be viewed by adding the data directly to the source of the img tag like this:
Since in contrast to the other services we are not dealing with JSON data here in the return message, there will be no error message if something goes wrong. Instead you will get an HTTP 400, 403, 404 or 500 So far only images are supported and no files such as x-rays. This issue will be addressed in further releases.
