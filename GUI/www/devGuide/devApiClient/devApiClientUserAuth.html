Client API for authentication


The authentication chapter on the client API is rather short. The login procedure is dealt with independently of the client API by the login page. All the request who are posed by a user who is not logged in will direclty leed to a redirecting to the login page. In other words, so far you can not use any kind of login procedure via the client API. But the API does support a query to check if the user is still logged in and a log out functionality. These will be described in the following.



Check login

This function allows the caller to check if the session key together with the users credentials is still active or if the user has to log in again. The function does not require any parameters and will not return. When the user is still logged in, nothing will happen. If not, the user will be redirected to the login page.
