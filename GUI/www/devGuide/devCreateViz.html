Getting Started in Building a Visualization for Health Bank©


Let us start by giving you a short introduction about the purpose of visualizations in the Health Bank© system.
A visualization should allow the user to illustrate data he or she has already in our system. For a running app one could think of a visualization that illustrates an aggregation of the latest entries or shows a map of the last track. Visualizations are displayed alongside the spaces pages in the Health Bank© system in so called dashboards. Users first have to install visualizations and cann then add currently only one of them to a space. Visualizations shall also have the possibility to illustrate data of multiple users, namly of all the users the currently logged in user has access to. This stays in contrast to the applications which are only allowed to add new data to the current user.



A visualization on Health Bank© is usually displayed within an iFrame. This iFrame is usually placed as a so called dashboard on top of the entries list in the different spaces. It consists of an HTML file and if needed an additional CSS and JavaScript file. To access our client API all it needs is a call to 'parent' within JavaScript code e.g parent.addRecord(). A step by step introduction on how this might look like is given in the demos section.
