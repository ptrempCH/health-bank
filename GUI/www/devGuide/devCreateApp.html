Getting Started in Building an Application for Health Bank©


Let us start by giving you a short introduction about the purpose of applications in the Health Bank© system.
An application in this system shall allow users to integrate services they know such as the mobile running app Runtastic as well as interesting new services like keeping track of your drug consumption. Their purpose is to gather information and store it on the Health Bank© system so that it can be used for complex queries. For the user the main benefit is that its data is not spread across dozens of different platforms but under his control here in the Health Bank© system.

There are actually two different ways of how to write an application for Health Bank©. One way is to extend the functionality of our page by adding interesting features like the example with the drug register. Applications like this run on our server and are actively used by the users of our system. The second approach for an app is that on our server there is just the set up and the actual app runs on the companies server. As an example on how this might work let us take a company that runs a mobile step counter application. This application is available in the different stores and counts the number of step a person takes each day. To integrate their data to our system that company can build an app that helps the user to set up the connection to the companies website by logging in its account for example. Additionally the user can select a few settings which will be stored on the companies website. To push the data finally to us the company will use our REST API to directly push data to the users account via the users ID and the app secret we created for them (Details on auth token see Authentication).



An application on Health Bank© is usually displayed within an iFrame. It consists of an HTML file and if needed an additional CSS and JavaScript file. To access our client API all it needs is a call to 'parent' within JavaScript code e.g parent.addRecord(). A step by step introduction on how this might look like is given in the demos section.



Here is the detailed procedure for an application that works without the need of a user interaction on our platform:

  1. An institute uploads a new application to the Health Bank© system.
  2. The system produces an app ID and an app secret which are only visible for the author of the application.
  3. A user installs the application. Hence the author of the app gets to know the userID and assigns it to the user on its server.
  4. The user produced some data on the institutes server (e.g. Runtastic).
  5. The institute calls the Health Bank© token REST request with the app secret, appID and userID and gets a access token specific for the user and application.
  6. The institute can now call the addRecord request with the apppD, userID and access token without the need for the user to log in first.
